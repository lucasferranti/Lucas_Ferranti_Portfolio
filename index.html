Projet M2 — Tolérance aux fautes d’un CNN quantifié INT8 (LeNet-5)
Étude approfondie de la robustesse d’un réseau convolutionnel embarqué face aux inversions de bits dans un contexte critique (systèmes embarqués sensibles aux radiations / défauts mémoire).

Contexte & Objectif

  • Analyse de la résilience d’un CNN LeNet-5 quantifié INT8 (dataset MNIST)
  • Simulation de défauts matériels via injection logicielle de bit-flips
  • Identification des couches et paramètres les plus vulnérables

Réalisations techniques

  • Développement d’un framework d’injection en C avec macro INLINE_BITFLIP
  • Instrumentation ciblée : poids, biais, activations intermédiaires
  • Automatisation complète des campagnes via Python
  • Exécution de +3 millions d’inférences
  • Analyse statistique par couche, bit et direction d’inversion

Résultats clés

  • Sensibilité élevée des bits de poids fort (MSB)
  • Couches C1 et C3 identifiées comme les plus vulnérables
  • Biais plus critiques que les poids

Conception de circuit numérique — Filtre passe-bas
Conception et validation d’un filtre passe-bas numérique : architecture à additionneur, multiplieur et retard z⁻¹, avec étude, simulation, synthèse et étapes de placement/routage.

Architecture & théorie

  • Déduction de la relation en Z et de l’architecture du filtre passe-bas (addition + multiplication + z⁻¹)
  • Spécification d’un coefficient de filtrage (ex : cm = 0,75 via 3092/4096)

Conception des blocs

  • Conception d’un additionneur n bits (propagation de retenue) à partir d’additionneurs 1 bit
  • Intégration filtre : registre d’entrée (bascules D), additionneur, multiplieur, retard z⁻¹

VHDL, simulation & validation

  • Description VHDL modulaire : registre (reg_E), additionneur avec registre (add_reg), multiplieur (multi), top filtre
  • Simulation comportementale + post-synthèse + post-routage (prise en compte délais)
  • Analyse des chronogrammes (mode analogique) : réponse progressive typique d’un passe-bas

Implémentation physique

  • Placement/routage avec Innovus : floorplan, alimentation (rings/rails), arbre d’horloge, fillers, export netlist/délais

SCADA / Automatisme — Modbus TCP + HOME I/O + Supervision
Mise en place d’une chaîne industrielle complète : commande PLC, simulation de partie opérative, supervision SCADA et analyse réseau des échanges Modbus TCP.

Architecture & objectifs

  • Contrôle d’équipements simulés (volets roulants) via programme PLC
  • Interfaçage PLC ↔ simulateur HOME I/O via passerelle I/O
  • Supervision SCADA : visualisation temps réel sur tableau de bord
  • Validation protocole : capture et analyse Modbus TCP sous Wireshark

Contributions techniques

  • Configuration réseau Ethernet : adressage IP, masques, tests de connectivité (ping)
  • Mapping I/O : correspondance entrées/sorties et zones mémoire (%I, %Q, %M, %MW)
  • Programmation Ladder : logique montée/descente avec exclusion mutuelle (sécurité)
  • Configuration SCADA : création équipement Modbus, trames de lecture/écriture, variables & synoptiques

Résultats & validation

  • Voyants SCADA cohérents avec les sorties PLC (réaction temps réel)
  • Analyse Modbus TCP : requêtes/réponses conformes, MBAP, fonctions de lecture/écriture
  • Mesure de performances : latence inter-trame stable (ordre de grandeur ~ dizaines de ms)
